SPT — Shifted Patch Tokenization

Each patch token is enriched with 4 shifted neighbours before projection, providing local context without relying on attention.

LSA — Locality Self-Attention

Full bidirectional attention (no causal mask) with a learnable per-head temperature and a Gaussian distance bias that softly prefers nearby patches.

LayerScale

Per-channel scalar γ on each residual branch, initialised at 1e-4. Stabilises early training on small datasets.

DropPath (Stoch. Depth)

Rate increases linearly across blocks (0 → 0.05). Acts as an ensemble of shallower networks at training time.

Per-Head Locality Bias

Each head has its own learnable locality_weight scalar (init 0.1), applied to the normalised Gaussian bias [−1, 0]. Heads can independently learn how local or global to be, producing the diversity that multihead attention is designed to exploit. The low init keeps the bias weak early in training so random QKV projections drive head divergence first.

Padding & Crop

457 is not divisible by 16. Reflect-pad to 464 before tokenisation, then crop back to 457×457 after reconstruction.

// Prerequisites & Setup
Required
🐍 Python 3.13.12
📦 Poetry 2.3.2
Notes
Code is optimized for execution on CPU-only HPC. Modification will be needed for CUDA acceleration.
HPC job submission uses the PBS workload manager.
1
Install dependencies
$ cd <repository-name>
$ poetry install
2
Activate virtual environment & run
Option A — activate shell
$ poetry shell
$ python main.py
Option B — run directly within virtual environment
$ poetry run python main.py
3
HPC execution (PBS workload manager)
optional
Modify script.pbs to match your cluster's resource requirements, then submit the job:
$ qsub script.pbs
⚠  Code is optimized for CPU-only HPC. Additional changes are required to enable CUDA acceleration.
